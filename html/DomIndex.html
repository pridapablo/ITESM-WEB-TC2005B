Preguntas

  1. ¿Qué es el DOM?
    Respuesta de ChatGPT: El DOM (Document Object Model) es una representación en forma de árbol de los elementos que componen un documento HTML o XML. Es una interfaz de programación que permite a los desarrolladores interactuar y manipular la estructura, contenido y estilo de un documento a través de lenguajes de programación como JavaScript.
    Respuesta de Copilot: El DOM (Document Object Model) es una interfaz de programación de aplicaciones (API) que representa una página web como un árbol de objetos. Cada objeto en el árbol representa un elemento del DOM, como un elemento HTML, atributo, texto, etc. Los objetos del DOM pueden tener propiedades y métodos que permiten manipular y navegar la estructura del DOM.
  2. ¿Qué es un nodo?
    Respuesta de ChatGPT: Un nodo es un elemento básico de la estructura del DOM. Cada objeto en el DOM, como elementos HTML, atributos, eventos y texto, es un nodo. Los nodos pueden tener nodos hijo y nodos padre, lo que permite navegar y manipular la estructura del DOM.
    Respuesta de Copilot: Un nodo es un objeto que representa un elemento del DOM. Los nodos se pueden clasificar en dos tipos: nodos de elemento y nodos de texto. Los nodos de elemento representan elementos HTML, como `p`, `div` y `img`. Los nodos de texto representan el contenido de los elementos, como el texto de un párrafo o el valor de un input.
  3. ¿Cómo se accede a un elemento del DOM?
    Respuesta de ChatGPT: Puedes acceder a un elemento del DOM utilizando métodos de selección como `getElementById`, `getElementsByClassName`, `getElementsByTagName`, y `querySelector` o `querySelectorAll`. Estos métodos se utilizan para seleccionar y almacenar referencias a los elementos del DOM que se deseen manipular.
    Respuesta de Copilot: Para acceder a un elemento del DOM, primero debes seleccionarlo. Para esto, puedes utilizar uno de los siguientes métodos:
    • getElementById
    • getElementsByClassName
    • getElementsByTagName
    • querySelector
    • querySelectorAll
    Por ejemplo, para seleccionar un elemento con el ID `my-element`, puedes utilizar el método `getElementById`: 
                  const element = document.getElementById('my-element');
  4. ¿Cómo se modifica un elemento del DOM?
    Respuesta de ChatGPT: Para modificar un elemento del DOM, primero debes acceder a él utilizando uno de los métodos de selección mencionados anteriormente. Luego, puedes cambiar sus atributos, contenido y estilo utilizando las propiedades y métodos del objeto seleccionado. Por ejemplo, puedes cambiar el contenido de un elemento con la propiedad `innerHTML` o agregar y eliminar clases con `classList`.
    Respuesta de Copilot: Para modificar un elemento del DOM, primero debes seleccionarlo. Para esto, puedes utilizar uno de los siguientes métodos:
    • getElementById
    • getElementsByClassName
    • getElementsByTagName
    • querySelector
    • querySelectorAll
    Por ejemplo, para seleccionar un elemento con el ID `my-element`, puedes utilizar el método `getElementById`: 
                  const element = document.getElementById('my-element');
    Una vez seleccionado el elemento, puedes modificarlo utilizando sus propiedades y métodos. Por ejemplo, para modificar el texto de un elemento, puedes utilizar la propiedad `textContent`: 
                  element.textContent = 'Nuevo texto';
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?
    Respuesta de ChatGPT: Los métodos `getElementById`, `getElementsByClassName` y `getElementsByTagName` son específicos para buscar elementos por su ID, clase o nombre de etiqueta, respectivamente. Por otro lado, `querySelector` y `querySelectorAll` son más versátiles, ya que permiten buscar elementos utilizando selectores de CSS. `querySelector` devuelve el primer elemento que coincide con el selector, mientras que `querySelectorAll` devuelve una lista de todos los elementos que coinciden.
    Respuesta de Copilot: La diferencia entre `getElementBy*` y `querySelector*` es que `getElementBy*` devuelve un elemento, mientras que `querySelector*` devuelve una lista de elementos. Por ejemplo, si se utiliza `getElementByClassName` para seleccionar un elemento con una clase específica, se devolverá el primer elemento que coincida con la clase. Si se utiliza `querySelectorAll` para seleccionar un elemento con una clase específica, se devolverá una lista con todos los elementos que coincidan con la clase. 
                  // Devuelve el primer elemento con la clase "my-class"
              const element = document.getElementsByClassName('my-class')[0];

              // Devuelve una lista con todos los elementos con la clase "my-class"
              const elements = document.querySelectorAll('.my-class');
  6. ¿Qué es un evento?
    Respuesta de ChatGPT: Un evento es una acción o suceso que ocurre en el navegador, como un clic, un cambio en un formulario, la carga de una página, etc. Los eventos pueden ser generados por el usuario o por el sistema, y los desarrolladores pueden usar eventos para interactuar y responder a acciones específicas en la página.
    Respuesta de Copilot: Un evento es cualquier acción que sucede en el navegador, como clickear un botón, escribir en un input, o hacer scroll en la página. Los eventos se pueden clasificar en dos tipos: eventos de usuario y eventos del navegador. Los eventos de usuario son acciones que realiza el usuario, como clickear un botón o escribir en un input. Los eventos del navegador son acciones que realiza el navegador, como cargar una página o hacer scroll en la página.
  7. ¿Qué es un listener?
    Respuesta de ChatGPT: Un listener (o escuchador) de eventos es una función que espera y responde a eventos específicos en un elemento del DOM. Los listeners se pueden agregar a elementos del DOM utilizando métodos como `addEventListener`. Cuando se dispara el evento, el listener ejecuta una función (callback) para manejar y responder al evento.
    Respuesta de Copilot: Un listener es una función que se ejecuta cuando se dispara un evento. Por ejemplo, si se agrega un listener al evento `click` de un botón, la función se ejecutará cuando se haga clic en el botón. 
                    const button = document.querySelector('button');

                button.addEventListener('click', () => {
                  console.log('Se hizo clic en el botón');
                });
    En el ejemplo anterior, la función se ejecutará cuando se haga clic en el botón.
  8. ¿Qué es un callback?
    Respuesta de ChatGPT: Un callback es una función que se pasa como argumento a otra función y se ejecuta en un momento específico, como cuando se completa una tarea o se produce un evento. En el contexto de los eventos, los callbacks se utilizan para definir el comportamiento que se ejecutará cuando se dispara un evento.
    Respuesta de Copilot: Un callback es una función que se pasa como argumento a otra función. Por ejemplo, si se agrega un listener al evento `click` de un botón, la función se ejecutará cuando se haga clic en el botón. 
                    const button = document.querySelector('button');

                button.addEventListener('click', () => {
                  console.log('Se hizo clic en el botón');
                });
    En el ejemplo anterior, la función se ejecutará cuando se haga clic en el botón.
  9. ¿Qué es un objeto de evento? Respuesta de ChatGPT: Un objeto de evento es un objeto que se crea automáticamente cuando se dispara un evento. Contiene información sobre el evento, como el tipo de evento, el elemento en el que se originó el evento y las propiedades específicas del evento. Los objetos de evento se pasan como argumento a las funciones callback de los listeners de eventos, permitiendo a los desarrolladores acceder y utilizar la información del evento en sus funciones.
    Respuesta de ChatGPT: Un objeto de evento es un objeto que contiene información sobre el evento que se está manejando. Por ejemplo, cuando se dispara el evento `click`, el objeto de evento contiene información sobre el elemento que generó el evento, la posición del mouse, etc. 
                    const button = document.querySelector('button');

                button.addEventListener('click', (event) => {
                  console.log(event);
                });
    En el ejemplo anterior, la función se ejecutará cuando se haga clic en el botón, y el objeto de evento se pasará como argumento a la función.

Conclusiones

Experiencia general

Mi experiencia general al realizar esta actividad utilizando las diferentes herramientas de AI fue muy interesante. Por un lado, me di cuenta que el código bien documentado o "boilerplate" es muy fácil de hacer usando copilot y ChatGPT.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

En general, creo que copilot es más útil para realizar ediciones básicas directamente en código (como crear una función), o replicar líneas de código con cambios secuenciales o que siguen algún patrón básico. Me imagino (por ejemplo) que copilot apresuraría mucho la creación de varios condicionales lógicos dentro de una función o un switch case.

Por otro lado, considero que ChatGPT es más útil para comprender documentación o para hacer cambios más complejos (como migrar de una librería a otra). Esto se debe a que en el prompt podemos dar contexto (como copy pastear documentación) para hacerla más entendible e incluso para realizar los cambios directamente en el códgio.

Ventajas de usar AI para programar

Considero que la inteligencia artificial puede ser una gran herramienta para programadores principiantes para comprender mejor ciertos conceptos (en el caso de ChatGPT) o para realizar tareas repetitivas (en el caso de copilot). Además, creo que la inteligencia artificial puede ayudar a los programadores más experimentados a realizar tareas de manera más rápida y eficiente.

Desventajas de usar AI para programar

Personalmente creo que el problema es que la inteligencia artificial no suele estar actualizada con las últimas convenciones de estilo ni con la documentación más reciente. Esto puede generar problemas de calidad en el código que se genera. Un ejemplo concreto, es que, tanto copilot como ChatGPT, generaron código con declaraciones de variables "var" en lugar de "let" o "const".

Por otro lado, creo que la inteligencia artificial no puede ser tan flexible como un humano en cuanto a análisis de contexto. Esto aplica especialmente para copilot, dado que no se le puede dar contexto más que a través del mismo documento donde se está escribiendo el código (a traves de comentarios).

Por último, creo que la inteligencia artificial es una herramienta que debemos aprender a usar para comprender problemas más complejos y para mejorar nuestra productividad, sin permitir que nos haga perder el control y entendimiento de nuestro código por intentar emplearla para resolvernos los problemas.
